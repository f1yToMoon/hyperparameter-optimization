# Отчёт о выполненной работе

Реализована лабораторная работа по теме **«Плохоформализуемые задачи на примере
подбора гиперпараметров деревьев решений»** согласно техническому заданию
(`# Техническое задание.md`).

## Результат

Один файл-деливерабл: **`Лабораторная_подбор_гиперпараметров.ipynb`** (20 ячеек,
выполнен целиком — все выводы и графики встроены).

## Что сделано по пунктам ТЗ

| Пункт ТЗ | Реализация в ноутбуке |
|---|---|
| 1. Определение плохоформализуемой задачи | Markdown-раздел с определением и всеми 5 свойствами (нет аналитики, нет градиента, дорогая оценка, шум, экспоненциальное пространство). |
| 2–3. Формализация | Введены $\lambda$, $A(\lambda)$, $w^*(\lambda)$, $Q(\cdot)$; записаны $F(\lambda)=Q(A(\lambda))$ и $\lambda^*=\arg\max_{\lambda\in\Lambda}F(\lambda)$. |
| 4. Обоснование black-box | Композиция «выбор → обучение → метрика», $\nabla_\lambda F$ не определён, дискретность/шум; **эмпирический график разрывности** $F$ по `max_depth`. |
| 5. Эксперимент | Датасет `breast_cancer` (sklearn), модель `DecisionTreeClassifier`, единое пространство `max_depth / min_samples_split / min_samples_leaf / criterion`, оценка = средняя Accuracy по 5-fold Stratified CV. |
| 6.1 Grid Search | Огрублённая сетка, полный перебор, формула сложности $O(n^k)$ + комментарий о неэффективности. |
| 6.2 Генетический алгоритм | Хромосома = вектор гиперпараметров; реализованы генерация популяции, fitness, турнирный отбор, равномерный кроссовер, мутация, элитизм; сложность $O(T\cdot N\cdot C)$. |
| 6.3 Bayesian Optimization | Optuna + TPESampler; objective, search space, фиксированный `n_trials`; описаны суррогат $\hat F$, acquisition через $l(\lambda)/g(\lambda)$, сложность $O(T\cdot C)$. |
| 7. Сравнительный эксперимент | Все методы на одинаковых данных/модели/пространстве; зафиксированы best score, время, число оценок функции (единый счётчик `N_EVALS`). |
| 8. Результаты в .ipynb | Теория + практика (3 метода) + таблица сравнения + графики «качество vs число итераций» и «время vs качество». |
| 9. Вывод | Сформулированы все требуемые тезисы о black-box природе и преимуществе BO/ГА над полным перебором при ограниченном бюджете. |

## Ключевые детали реализации

- **Единая целевая функция** `objective(params)` — общий «чёрный ящик» для всех трёх
  методов, что обеспечивает корректность сравнения.
- **Единый счётчик оценок** `N_EVALS` считает фактическое число обучений модели
  (вызовов $F(\lambda)$) — главная метрика бюджета вычислений.
- **Фиксированный `RANDOM_STATE = 42`** для воспроизводимости.
- Вспомогательные графики (разрывность) не учитываются в бюджете методов.

## Полученные результаты (последний прогон)

| Метод | Best score | Оценок F | Время, c |
|---|---|---|---|
| Grid Search | 0.9385 | 192 | 3.80 |
| Генетический алгоритм | 0.9385 | 108 | 2.09 |
| Bayesian Optimization (TPE) | 0.9385 | **60** | **1.22** |

**Вывод эксперимента:** все три метода достигают одинакового качества, но
байесовская оптимизация и генетический алгоритм доходят до него существенно меньшим
числом обучений модели, чем полный перебор, — что подтверждает их эффективность для
black-box задач при ограниченном бюджете вычислений.

в `HOWTO.md`.
